Iaas : 고객에게 서버, 네트워크, os, 스토리지를 가상화하여 제공하고 관리함.
        os, 미들웨어, 애플리케이션 및 데이터와 같은 자원들을 손수 관리해줘야 함.
	VM, Server, Storage, Networking
paas : 고객에게 os, 미들웨어, 런타임과 같은 소프트웨어 작성을 위한 플랫폼을 가상화하여 제공
         가상화된 플랫폼은 웹을 통해 제공됨 그리고 개발 툴을 지원하기도 함 = 개발에 집중 ㄱㄴ
         * OS에 대한 액세스를 제공하지 않음 * 자동확장(automatically) 기능 제공
         *개발자가 클라우드 기반 애플리케이션을 개발하거나 사용자 지정하기 위해 구축 할 수있는
           프레임 워크를 제공합니다.
	O/S, Middleware, Rintime 외에 Iaas 기능들
	*Iaas 기능을 포함하고 있음.
saas : 고객을 대신하여 소프트웨어와 데이터를 제공하고 관리함. 개별 컴퓨터에 응용프로그램을 설치할
        필요가 없다. 최소한의 관리만 필요함. 현재는 공급자가 모든 책임을 짐.
        ex)전자 메일, 일정 및 Office 도구
	*Iaas, Paas 기능을 포함하고 있음.

IIS : 마이크로소프트에서 제공하는 VM 사이트

*자동 확장 : 웹 앱을 호스팅하기 우해 부하 분산된 가상 머신을 더 추가하는 것(이나 줄이는 것)

옛날에 IT는 CapEx(미래 이윤을 창출하기 위해 지출된 비용)였으나 
오늘날 클라우드 시스템은 OpEx(운영 비용)으로 전환되고 있다.

*가상머신(VM)
가상 머신의 세팅은 다를 수 있다.
= 월펼 비용이 다를 수 있다.
가상머신은 사용한 만큼의 비용을 지불한다. 
하지만, 가상 머신과 관련된 스토리지 비용은 계속 지불해야한다.(가장 일반적으로 디스크 비용)

*내결함성 : 일부 구성 요소의 장애 발생시 계속해서 작동할 수 있도록 해주는 시스템 기능

pubilc cloud : 사설 클라우드로 데이터 센터에서 호스팅된다. 가장 일반적인 배포 모델로 
모든 것이 클라우드 공급자의 하드웨어에서 실행됩니다. 
대표 기업 : AWS ,MS Azure, IBM Cloud
특징 : 측정 가격, 셀프 서비스 관리

private cloud : 자신만의 클라우드 시스템을 구축하는 것을 말한다. 
                    이는 주로 자신들만의 정보망을 구축하기 위해 기업에서 많이 사용한다.

Hybrid cloud : 단일 IT 환경처럼 보이지만, 실제로는 여러 환경이 연결된 형태
애플리케이션을 개별적이지만 연결되어 있는 다중 환경 내외로 이동할 수 있는 경우 하이브리드 클라우드라고 볼 수 있다.
